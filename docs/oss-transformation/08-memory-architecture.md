# Deliverable 8 ‚Äî Memory Architecture

Seven memory tiers under `.memory/`, all plain YAML/Markdown (git-versioned, platform-neutral, diff-reviewable). Materializes `ai-delivery-os/05-memory-architecture.md`.

## 1. Tier Map

| Tier | Path | Scope | Written By | Retention |
|------|------|-------|-----------|-----------|
| Strategic | `.memory/strategic/` | Organization-wide principles, tech radar | Governance agent (gated PR) | Permanent, versioned |
| Domain | `.memory/domain/<domain>/` | Business domain knowledge, glossaries, regulations | Analyst | Permanent, per-domain |
| Pattern | `.memory/pattern/` | Reusable architecture/model patterns (e.g. "four-eyes approval subprocess") | Architect | Permanent; promotion pipeline below |
| Decision | `.memory/decision/` | ADR index across all projects | Architect/Governance | Permanent, immutable entries |
| Project | `.memory/project/<project>/` | Project context, stakeholders, constraints | PO | Archived at project close |
| Session | `.memory/session/<session-id>/` | Current execution context, working notes | Any agent | Compressed at session end; purged after harvest |
| Failure | `.memory/failure/` | Lessons learned, anti-patterns, skill gaps | QA/Governance | Permanent, indexed by category |

## 2. Required Operations

### Indexing
`.memory/index.yaml` ‚Äî master index: every entry gets `id`, `tier`, `tags`, `summary` (‚â§2 lines), `path`, `updated`. Rebuilt by `.tools/memory-index` (CI verifies freshness). Agents load the index (cheap) and fetch entries on demand (targeted).

### Retrieval
Protocol (in every agent's instructions): (1) read index ‚Üí (2) match tags against current task (domain, stage, model type) ‚Üí (3) load top-N entries by relevance ‚Üí (4) cite loaded memory IDs in produced artifacts (feeds trace chain).

### Compression
Session memory is append-only during execution. At session end, the active agent runs the **harvest step**: distill the session log into ‚â§20-line summaries routed to the correct durable tier; raw session files are then eligible for purge.

### Summarization
Every entry carries its own `summary` front-matter. Tier-level `SUMMARY.md` files (regenerated by `.tools/memory-index`) give agents a one-page view per tier ‚Äî this is what fits in always-on context.

### Versioning
Git is the version store. Strategic/Decision tiers additionally carry `version:` and `supersedes:` fields; entries are never edited destructively ‚Äî superseding entries link back.

## 4. Memory‚ÄìAgent Binding

Each agent charter declares `memory: { reads: [...], writes: [...] }`. Examples: `dev` reads pattern+project+session, writes session only; `governance` is the sole writer to strategic. CI lints that writes in a PR match the author agent's declared scopes (via PR labels).

## 5. Privacy and OSS Hygiene

`.memory/project/` and `.memory/session/` are gitignored by default in downstream usage (template repo ships examples only); contributors never commit client-identifiable data ‚Äî checked by a CI secret/PII scanner.
