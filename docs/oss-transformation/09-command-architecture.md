# Deliverable 9 — Command Architecture

Twelve canonical commands in `.commands/`, identical behavior across Claude Code, OpenCode, Copilot Agent Mode, Cursor, and Codex via thin generated adapters.

## 1. Canonical Command Format

```markdown
# .commands/specify.md
---
id: specify
stage: 3-specification
agent: po
inputs: [discovery report, analysis report]
outputs: [.specs/<project>/requirements.yaml, stories/]
templates: [requirements, user-story, epic]
preconditions: [/discover completed, /analyze completed]
gate: G1-requirements (exit)
---
## Procedure
1. Load template `requirements` from .templates/registry.yaml (template-first rule).
2. Hydrate memory: domain/<domain>, pattern, failure tiers.
3. Elicit/derive requirements; every requirement gets id, source story, acceptance criteria.
4. Produce requirements.yaml with status: pending-approval.
5. Request human approval (gate G1). HALT until approved.
```

## 2. Command Catalog

| Command | Stage | Agent | Output | Gate |
|---------|-------|-------|--------|------|
| `/discover` | 1 | analyst | Discovery report: actors, goals, domain candidates | — |
| `/analyze` | 2 | analyst | Domain analysis: bounded contexts, glossary, best-practice research (OWASP/NIST etc.) | — |
| `/specify` | 3 | po | `requirements.yaml`, stories, epics | **G1 exit** |
| `/architect` | 4 | architect | `architecture.yaml`, ADRs, root-model selection (BPMN/CMMN/hybrid) | **G2 exit** |
| `/design` | 5 | architect | Data model, data dictionary (schema impact), API contracts | **G3 exit** |
| `/model` | 6 | dev | Flowable model YAML: App → Root → Dependent → Forms (16-state lifecycle) | — |
| `/review` | 7 | qa + governance | Review report, trace-chain audit | — |
| `/plan` | 8 | po | `tasks.yaml` (approved task breakdown) | task approval |
| `/build` | 9 | dev | Implementation artifacts | **blocked unless G1+G2+G3 + tasks approved** |
| `/test` | 10 | qa | Test results, golden-test runs, coverage evidence | feeds G4 |
| `/release` | 11–12 | governance | Release manifest, sealed trace chain | **G5 exit** |

(`/model` and `/build` are the only generative-implementation commands; both are hard-gated.)

## 3. Cross-Platform Adapter Matrix

| Platform | Adapter Location | Mechanism |
|----------|-----------------|----------|
| Claude Code | `.claude/commands/<id>.md` | Slash command wrapping canonical file |
| OpenCode | `.opencode/command/<id>.md` | Frontmatter `agent:` routing + file reference |
| Copilot | `.github/prompts/<id>.prompt.md` | `mode: agent` prompt file |
| Cursor / Codex / Gemini | `AGENTS.md` command index | Command table instructs reading `.commands/<id>.md` |
| BMAD | task files in BMAD adapter | Generated from canonical frontmatter |

All adapters are generated by `.tools/adapter-sync`; CI fails on drift.

## 4. Execution Contract (every command)

1. **Boot**: constitution → memory index → manifest (per Repository OS boot sequence).
2. **Precondition check**: verify required predecessor artifacts exist with `status: approved`; otherwise emit "blocked — run X first" and stop.
3. **Template selection**: resolve from `.templates/registry.yaml`; refuse to generate without one.
4. **Execute** the procedure; log state transitions to the execution log (Initialized → In Progress → … per generation lifecycle).
5. **Harvest**: write session learnings; update `.governance/trace/` links.

## 5. Consistency Guarantee

Behavioral parity is tested: `.tools/command-conformance` runs each command's golden scenario (login-page example) on CI against recorded transcripts and asserts identical artifact structure across platforms.
