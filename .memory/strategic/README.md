# Strategic Memory — OSS Governance Principles

1. **One-command experience** — clone → open in any AI agent → boot to production state.
2. **Spec-driven first** — no code before approved requirements, architecture, and tasks; enforced structurally.
3. **Single source of truth, thin adapters** — tool-neutral `.ai/` core; generated platform adapters.
4. **Template-first generation** — no AI output without template selection.
5. **Human-in-the-loop is non-negotiable** — Requirements, Architecture, Security, Data Model, Release approvals cannot be bypassed.
6. **Community as architecture** — contribution, governance, and growth are designed systems.

Targets: 5,000+ stars, 500+ forks, 100+ contributors. Roadmaps: `docs/oss-transformation/14-roadmap-5000-stars.md`, `15-roadmap-10000-stars.md`.
