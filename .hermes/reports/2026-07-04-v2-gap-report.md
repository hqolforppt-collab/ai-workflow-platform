## 4. Gap register

### Critical

**G1 — MCP mutating tools bypass governance (Constitution R2 violation).**
`packages/flowable-mcp-server/src/index.js` implements `deploy` (66–113), `start_process` (191–212), and `complete_task` (240–259) with **no G1–G4 gate check**. Any MCP client can deploy an ungated blueprint to a live engine. The CLI path does check gates (`flowable.js:131–137, 339–370`), so the logic exists but is not shared. Compounding: `checkGates` **permits deploy with a warning when gate files are absent** — the plan requires refusal unless all four gates are approved. Fail-open governance.

**G2 — MCP server unreachable by design.**
No `.mcp.json` exists; no client config or setup doc registers the server. As shipped, the MCP half of E3 is dead code. Root cause: plan defect P2.

**G3 — Zero tests for all v2.0 code; CI exercises only v0.1 paths.**
No `.test.js` / `.spec.js` anywhere in either package. CI (`validate.yml`, `bpmn-roundtrip.yml`) runs `awp validate`, `awp build --check`, and the two pre-existing scripts. Nothing touches `--execute`, `--staged`, `--resume-from`, the rule engine, tier resolution, `flowable convert/deploy`, or any MCP tool. Plan items F2.5, F3.2, F3.5, F4.2 unmet. G4 is the proof of consequence.

### High

**G4 — `awp build --aggregate` is broken and was never run.** *(verified by execution)*
Crashes with `require is not defined`: `mergeStagedFiles` (`build.js:270–279`) uses CommonJS `require()` inside an ESM package (`"type": "module"`). Even the intended logic is wrong — it joins files with `\n---\n` (multi-document YAML) instead of the plan's shallow section union with `workflows = {...root, ...dependent}` and unique-id assertion (§7.5). Monolith consumers would receive an unparseable-as-one-doc file.

**G5 — Deterministic validator not exposed as a CLI command.** *(verified by execution)*
`awp validate <blueprint-dir>` runs the old v0.1 platform-OS check (spec/gates/adapters/memory) and ignores the argument. The 17-rule engine in `src/validate.js` runs only inside `build --execute`, so a manually repaired blueprint cannot be re-validated — breaking the plan's own `DRAFT-INVALID → repair → re-validate` loop (§5.4).

**G6 — Converter coverage is shallow.** *(verified by execution)*
`awp flowable convert` on the golden example emitted **1 artifact** (`root.bpmn20.xml`, generic key `root`, + manifest). Plan target for this exact example: ~5 artifacts (root + 2 dependent BPMN, 1 form.json, 1 DMN — §8.4). Forms, dependent workflows, and decision tables were silently dropped.

**G7 — Skill `mcp:` delegation blocks missing (F3.4 / §8.5).**
All five skills (`bpmn-`, `cmmn-`, `dmn-`, `form-modeling`, `root-model-selection`) lack the planned `mcp:` blocks; the authoring flow never invokes `validate_bpmn` / `get_form` / `list_processes`.

**G8 — Staged golden example missing (F4.3).**
`examples/workflow-builder/login-registration/` contains only the monolithic `blueprint.yaml`; no `01–06-*.yaml`, no `flowable/` output; `golden-tests.yaml:8` references the monolith only. No reference artifact exists for the format the whole plan is built around.

### Medium

**G9 — Rule-count inconsistency + stub rules.** "16 rules" appears in plan, `prompt.md`, `pipeline.yaml`, `command.yaml`; the file has 17. `VAL-032` and `VAL-040` are unconditional passes (`validate.js:293–298`).

**G10 — `zod` missing from `awp-cli`** (plan §6.4, Appendix B). Validation uses ad-hoc regex/manual checks — works, but diverges and weakens shape guarantees.

**G11 — No `.awp/config.example.yaml`** or any config template (F1.4; root cause P3).

**G12 — No schema-summary sync golden test** (F1.3 acceptance criterion: every `required: true` field in `schema.yaml` appears in the summary).

**G13 — Documentation lag (F4.4).** README, CHANGELOG, AGENTS.md, `.commands/DISCOVERY.md`, and `command.yaml` contain no v2.0 flags/features; CHANGELOG has no v2.0 entry despite the commit message claiming v2.0.

**G14 — MCP server layout diverges.** One ~293-line monolith instead of `src/client.js` + `src/tools/*` + `test/`. Cosmetic on its own, but it is part of why the tools are untestable and the gate check was not shared.

### Low

- `--aggregate` missing from CLI help text (`index.js:14`).
- Tier `match` globs drift slightly from the plan (extra `qwen*` / `deepseek*` entries — harmless, arguably an improvement).
